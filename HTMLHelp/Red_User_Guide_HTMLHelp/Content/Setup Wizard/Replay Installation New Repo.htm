Replay Installation - New Repository

  1. Selecting the Create new repository option displays the Configure Metadata Database screen that enables you to specify a data source to which RED connects and the log in details for the selected source database.


    Type in the required metadata connection details:
    FieldDescription
    Data Source NameThe ODBC data source name (DSN) for the metadata repository connection.
    UsernameThe log in user name to connect to the metadata repository connection.
    PasswordThe password for the specified user name.
     
  2. Click Validate to continue the data source and log in credentials validation.
    Notes:
    The validation is performed against the selected data source to ensure:
    • The database is not already being used by another RED repository.
    • The database is not being used for another purpose.
  3. After successful validation, click Next to continue. The Load Installer Settings window is displayed and prompts you to select a file that contains RED Setup Wizard settings from a previous installation.
    Click Browse to locate and select the settings file you want to use with the RED Setup Wizard.

  4. Click Next to continue, the Configure Repository Settings window is displayed with options on how to configure the RED metadata.


    Select a repository setting configuration option:
    OptionDescription
    Default settingsSelect this option to configure your repository with default settings.
    Import settings from fileSelect this option to browse to a directory and import a file that contains the repository settings you want to use.
    Copy settings from repositorySelect this option to connect to an existing repository and copy its settings. You must specify the Data Source Name and the associated login credentials for validation.
  5. Click Next to continue, the Enable Databases window is displayed with a list of Enablement Packs retrieved from the WhereScape website that are associated with the defined metadata database.
  6. Click Next to continue, the Configure Enablement Packs window is displayed, showing a list of components that you can include or exclude in RED
    .

    All components are selected by default. Click the drop-down arrow to view a sub-menu of the items available under each component.
    Note:
    The enablement pack components vary, depending on the type of database enabled.
    OptionDescription
    Database Function SetEnables you to select the RED database function set that will be installed with the enablement pack.
    Database function sets contain a list of functions and operators that are used for building transformations.
    Data Type Mappings SetEnables you to select the RED data type mapping set that will be installed with the enablement pack.
    Data type mapping sets contain a list of mappings that are used when loading tables into the data warehouse.
    Extended PropertiesEnables you to select the RED extended property definition set that will be installed with the enablement pack.
    Extended properties are used to facilitate connection to a specific database technology (e.g. Snowflake, Amazon Redshift, Microsoft Azure, etc.) and also used in the script based processing of all table objects in the database target.
    TemplatesEnables you to select the RED templates that will be installed with the enablement pack.
    Templates are used in processing the data warehouse table objects. .
    ScriptsEnables you to select the RED scripts that will be installed with the enablement pack.
    Scripts are used in processing the data warehouse table objects. .
    ProceduresEnables you to select the RED procedures that will be installed with the enablement pack.
    Procedures are used in processing the data warehouse table objects. .
    Deployment ApplicationsEnables you to select the RED applications that will be installed with the enablement pack.
    Deployment applications consist of set of RED data warehouse objects that are shipped to enable inclusion of those objects in a metadata repository
    Files Automatically ExecutedEnables you to select the RED files that will be installed with the enablement pack.
  7. Click Next to continue, the Add Database Targets window is displayed.


    Type in the required target connection and schema details:
    FieldDescription
    Connection NameThe name used to label the target database connection within WhereScape RED. Typically Repository for target databases like Snowflake, Redshift, Greenplum, etc.
    Data Source Name (DSN)The ODBC data source name (DSN) for the target database connection. The OBDC Data Source Name must be defined in the Windows ODBC Data Source Administrator to appear in this drop-down field.
    UsernameThe log in user name to connect to the target database.
    PasswordThe password for specified user name.
    SchemasEnables you to add a schema in the specified target database. If the schema defined does not exist in the target database, it is created during the installation process.
    AddEnables you to add a database schema. A list of the added targets are displayed in the right pane of the window.
    ValidateEnables you to validate the specified database schema.
  8. After successfully adding your database target(s) and schema, click Next to continue.
    The Add Sources window is displayed.


    Type in the required data source connection details:
    FieldDescription
    Connection NameThe name used to label the data source connection within WhereScape RED. Typically similar or derived from the ODBC Source Name (DSN) for quick identification.
    Data Source Name (DSN)The ODBC data source name (DSN) for the data source connection. The OBDC Data Source Name must be defined in the Windows ODBC Data Source Administrator to appear in this drop-down field.
    UsernameThe log in user name to connect to the data source.
    PasswordThe password for specified user name.
    ValidateEnables you to validate the specified data source.
    AddEnables you to add several data sources. A list of the added sources are displayed in the right pane of the window.
  9. After successfully adding and validating your data source(s), click Next to continue.
    The Configure Scheduler window is displayed.


    Type in the required scheduler details:
    FieldDescription
    Scheduler NameWhen scheduling a job with RED you can assign the job to a specific scheduler. Each scheduler must therefore have a unique name. The default name is WIN0001. Most users change this to be the scheduler machine's name followed by a number, e.g.: DWHP01, where DWHP is the name of the windows scheduler server.
    Scheduler IDThe unique name by which the scheduler service is referenced. Each scheduler must have a unique identifier (e.g. SID_user).
    Log DirectoryThe log directory used to store some temporary files created by the scheduler.
    Log File NameThe log file name where the scheduler writes information. The log file contains information on the scheduler service. This file may become large if log level 9 is set.
    Log LevelEnables you to define the level of detail written to the log file by the scheduler service. This must be left at the default level unless requested by WhereScape Support.
    The following logging levels are available:
    • Standard - select this option to only log major events
    • Debug mode - select this option to log full debug information
    Login ModeEnables you to login using a System Account or User Account.
    Selecting the User Account mode requires you to enter the following:
    • Username - log in user name for the user installing the scheduler service.
    • Debug mode - password for specified user name.
    Test userEnables you to test if the specified user has the right permissions to install and manage the scheduler service.
  10. Click Next to continue, the Pre-install Check window is displayed with a list of pre-installation requirements and their status. You can fix pre-installation items that failed and use the Re-check button to verify if they have been fixed.
    You can also select the Bypass pre-install checks option to proceed without performing the pre-install checks.

  11. Click Next to continue, the Summary window is displayed showing the details about the Metadata Location, Enabled Database and Scheduler settings.

  12. Click Install to proceed with the installation.
    Note:
    Clicking Previous enables you step back through the Setup Wizard screens and edit the settings you have currently defined.

    Once the installation is complete, a message is displayed to confirm that the installation was successful.
  13. Click Finish to launch RED and start working with your data warehouse.
    Note:
    If the installation process was not successful, a message is displayed showing the error details. You can try to fix the problem using the error information and then click Retry to continue the installation.
