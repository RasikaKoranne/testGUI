Metadata Upgrade Required

Notes:
Ensure the scheduler is not running and no users are accessing the metadata before doing an upgrade.

Upgrade of the metadata is required if the Metadata Validate tool identifies changes in the metadata version or table objects in the metadata repository. Each type of upgrade is described below.

Metadata version upgrade

  1. If the Metadata Validate tool determines that the metadata version is not the same as the WhereScape RED version installed, a window is displayed advising you that a metadata upgrade is required:


  2. Click Next to continue. The Validating Metadata window is displayed which lists the status of each table object inspected in the repository. A summary of the number of table objects inspected and actions required (if any) are also displayed below the list.
    Refer to Metadata objects upgrade below, if discrepancies in table objects have been identified.
  3. Click More info to display the Metadata Validate Report which provides a summary of the validation issues found and the corresponding recommendations. You can export the report to a file and save it to a specified directory, or copy it to the Windows clipboard if required.
  4. Click Close to return to the Validating Metadata window and then click Next to continue.
  5. The Upgrade metadata commands window is displayed which details the commands that will be executed to upgrade the metadata. The upgrade Options are all selected by default.
  6. Click Next to proceed with the metadata version upgrade. The Upgrading Metadata window is displayed which lists the status of each table object upgraded in the repository. A summary of the number of table objects inspected and actions performed are also displayed below the list.
  7. After completing the upgrade, a window is displayed to confirm successful validation of the metadata repository.
    Click Finish to close the Metadata Validate tool.

 

Metadata objects upgrade

  1. If the Metadata Validate tool determines that the metadata version is correct but the metadata table objects is not the same as the WhereScape RED version installed, then the Validating Metadata window is displayed and lists the status of each table object inspected in the repository. A summary of the number of table objects inspected and actions required (if any) are also displayed below the list.
  2. Click More info to display the Metadata Validate Report which provides a summary of the validation issues found and the corresponding recommendations. You can export the report to a file or copy it to the Windows clipboard if required.


  3. Click Close to return to the Validating Metadata window and then click Next to continue.
  4. The Upgrade metadata commands window is displayed which details the commands that will be executed to upgrade the metadata. The upgrade Options are all selected by default.
  5. Click Next to continue. The Upgrading Metadata window is displayed which lists the status of each table object upgraded in the repository. A summary of the number of table objects inspected and actions performed are also displayed below the list.
  6. After completing the upgrade, a window is displayed to confirm successful validation of the metadata repository.
    Click Finish to close the Metadata Validate tool.
