One to One Views

A one to one view is a database view of a dimension table. It may be a full or partial view. It is typically used to provide an access path to the dimension tables with a built in locking clause.

In many data warehouses views are built as part of the end user layer, but creating them in the data warehouse means they are available regardless of the end user tools used.

The process for creating a view is as follows:

  1. Double click the View group object in the left pane.
  2. Browse to the Data Warehouse in the right pane.
  3. Drag a table from the right pane into the middle pane.
    • The Add New Metadata object window displays and defaults the object type to a View table.
    • Change the view name as required, and click ADD.

    The View Properties window displays:

    l

    Change the following properties, as required:

    • Select the Distinct Data Select check box if you want the view to return only distinct values.
    • You can enter a From/Where clause, but this can be done later.
      Clicking the ellipses button opens the Source Join wizard shown below. Refer to Generating the Dimension Update Procedure > Source tab > Joining multiple source tables for details.

    The Table Locking Mode can be changed at this point. The default is LOCK ROW FOR ACCESS, all other Teradata locking modes are available for use.

  4. Click OK after completing the changes.

  5. A window displays indicating that the view has been defined. Click Create View to create the view in Teradata.
